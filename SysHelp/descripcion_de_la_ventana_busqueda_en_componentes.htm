¿Cómo realizar una búsqueda?
Para ello hay que seleccionar los datos que pide en cada parámetro y llenar los campos Busca, Query y Máscaras y finalmente dar clic en el botón .
La búsqueda de componentes permite localizar de manera ágil a aquellos que cumplen las características establecidas mediante parámetros.

Para realizar una búsqueda de componentes, es necesario especificar los siguientes parámetros en los que ésta se apoyará:
Sistema
Muestra el nombre del sistema actual en el que se quiere realizar la búsqueda.
 
Arriba
Clase
Muestra la lista de aquellas que están disponibles dentro de la base de conocimiento.
Para este campo se debe seleccionar un nombre de clase de las que se enlistan en el combo.    
 
Arriba
Bibliotecas
Muestra la lista de aquellas que están disponibles dentro de la base de conocimiento.
Para este campo se debe seleccionar un nombre de biblioteca de las que se enlistan en el combo.
 
Arriba
Busca
Texto a buscar en los componentes de la biblioteca seleccionada.
Se debe teclear la palabra que buscará con los siguientes símbolos:
 
+   La palabra debe existir en el fuente
-    La palabra NO debe existir en el fuente
Sin símbolo inicial (El fuente debe tener una u otra de las palabras)
@   Las palabras deben estar en la misma línea
<   La palabra no debe estar en la misma línea
 
 
Arriba
Query
Selecciona la informaciòn, en base al query que se especifique.
Se debe teclear un SQL formado bajo las siguientes condiciones:
-----
 
                 Campos reconocidos:
 
$SOURCE
                  Nombre del fuente
$WORD
                  Palabra sencilla
$DIVISION
                  Division de COBOL
$SECTION
                  Sección COBOL
$ROUTINE
                  Nombre de rutina COBOL
$LINE
                  Número de línea
$RECORD
                  Texto de la línea
$COMMAND
                  Nombre del comando
$PARAMS
                  Parámetros del comando
'CADENA'
                  Cualquier cadena de caracteres
 
En SELECT:
Permite opcionalmente el "AS" en los alias:
Select $source AS fuente, $command comand, $record as registro from $sources
 
En FROM:
Sólo reconoce como tabla $SOURCES
 
En WHERE:
Operadores =, >, <, OR, AND, Paréntesis para agrupar o dar prioridades
 
Ejemplos de uso de SQLs:
 
Ej.1 - select $source,$routine rutina, $command comando,$params parametros from $sources where $command='COMPUTE'
Ej.2 - select $source fuente,$routine rutina,$line linea, $command comando,'->' usa,$params parametros, $record registro from $sources where ($command='COMPUTE') or ($command='MOVE')
 
 
Uso del operador IN
Para buscar una palabra entre los parámetros de un comando:
SELECT $source,$command,$params,$RECORD FROM $SOURCES WHERE 'CTA-ANO-4' IN $PARAMS
Nota: En la columna $RECORD aparece la línea donde encontró el comando. (Operando en MODO  COMANDO)
 
Para desplegar las lineas donde aparezca una palabra:
SELECT $source,$command,$params,$RECORD FROM $SOURCES WHERE 'CTA-ANO-4' IN $RECORD
Nota: En la columna $RECORD aparece la linea donde encontró la palabra CTA-ANO-4 (al utilizar "IN $RECORD" se cambia la operación a MODO LINEA)
 
Uso de distinct
Select distinct $source, $routine,$command from $sources where $division='PROCEDURE'
 
Uso de GROUP BY
Select $source, $routine,$command,count(*) from $sources where $division='PROCEDURE' group by  $source, $routine,$command

Máscaras
Permite especificar letras iniciales ó  nombre completo de los  componentes  de la aplicación, que es de interés en la consulta.
En este campo se teclea el prefijo del nombre de los fuentes seguido de un asterisco.
 
Arriba
Botón Ejecutar
Una vez establecidas las características de la búsqueda, se puede solicitar ejecutar la consulta.

Relación de componentes encontrados
En este panel se puede encontrar la lista de elementos encontrados de acuerdo a las condiciones de búsqueda establecidos.
Cada elemento encontrado contiene una liga, al dar un clic sobre el mismo, permitirá en el panel superior derecho, conocer las líneas del código donde éste fue encontrado y en el panel inferior derecho el detalle del código fuente.

Relación de líneas encontradas
Especifica las líneas que dentro del código fuente, contienen los elementos especificados en las condiciones de búsqueda.
